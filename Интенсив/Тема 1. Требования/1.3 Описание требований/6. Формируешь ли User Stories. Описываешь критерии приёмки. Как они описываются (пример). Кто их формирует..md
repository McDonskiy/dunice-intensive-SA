_**User Stories** (пользовательские истории) это описание функций продукта, помагающих определить требования к функциональности системы с точки зрения конечного пользователя._

_**Формирование User Stories**_

_User Stories обычно формируются продуктовым владельцем (Product Owner) в сотрудничестве с командой разработки и заинтересованными сторонами (стейкхолдерами)._

_**Структура User Story**_

_Стандартная структура User Story выглядит следующим образом:_

> _Как <роль пользователя>, я хочу <действие/функционал>, для того чтобы <цель/результат>._
